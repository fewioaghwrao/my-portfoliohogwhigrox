給与明細管理アプリ

中小企業・小規模事業者を想定した 給与明細の管理・共有 に特化した Web アプリケーションです。
管理者は従業員マスタと給与データを登録するだけで、
月ごとの給与明細をブラウザ上から確認・ダウンロードでき、
従業員はマイページから自分の給与明細のみを閲覧できます。

デモサイト(一般ユーザー用ログイン) デモサイト(管理者用ログイン) GitHub リポジトリ

※現在はポートフォリオ用の検証環境として利用しています。
社内利用を想定したロール設計・バリデーションを重視しています。

主な機能

管理者機能

  • 従業員マスタ管理（登録・編集・退職フラグ）
  • 給与明細の登録・編集・削除
  • 年月・従業員・部署でのフィルタ検索
  • 支給合計・控除合計・差引支給額の自動計算
  • 給与明細の一覧出力（CSV or PDF を想定）

従業員機能

  • ログイン／ログアウト
  • 自分の給与明細一覧の閲覧
  • 各明細の詳細表示（支給・控除内訳）

権限・セキュリティ

  • 管理者／従業員ごとのロールベースアクセス制御
  • 従業員は自分の明細にのみアクセス可能
  • CSRF 対策・フォームバリデーション
  • 入力値のサーバーサイド検証（必須・数値範囲チェック等）

アーキテクチャ・DB設計

構成・責務分担

Laravel の MVC パターンに沿って設計しています。

  • Controller：ルーティング・認可チェック・入力値の受け渡し
  • FormRequest：バリデーションルール・メッセージ定義
  • Service（導入している場合）：給与計算や集計処理
  • Model（Eloquent）：テーブルとドメインモデルの対応
  • View（Blade）：画面レイアウト・入力フォーム
PayrollController
  ├─ index()      // 明細一覧
  ├─ create()     // 新規登録画面
  ├─ store()      // 登録処理
  ├─ edit()       // 編集画面
  └─ update()     // 更新処理

主なテーブル例

  • users：従業員ログインアカウント（社員番号・氏名・メール・部署・雇用区分など）
  • admins：管理者ログインアカウント（氏名・メールなど）
  • payslips：給与明細ヘッダ（従業員・年月・支給合計・基本給・税金など）

正規化しすぎず、実務でありがちな給与項目の増減にも対応しやすい構造を意識しています。
ER 図については GitHub リポジトリの docs/ 以下に配置しております。

実装上の工夫・テスト

実装面で意識したポイント

  • 給与計算に関わる数値項目は decimal 型で管理
  • 支給・控除区分ごとに小計を算出し、差引支給額を自動計算
  • 従業員が他者の給与明細を閲覧できないような権限制御
  • 給与に関するデータのため、誤操作防止のための確認ダイアログ・メッセージを配置
  • 一覧画面は年月・従業員などで検索できるように実装

テスト

  • FormRequest のバリデーションテスト
  • 給与明細登録・更新処理の単体テスト
  • 認可ロジック（管理者／従業員）のテスト
  • 一部画面の HTTP テスト（レスポンスステータス・リダイレクト先の確認）

テストコードは GitHub リポジトリの tests/Feature および tests/Unit に配置しています。
