請求書・入金ステータスダッシュボード（Lite）

請求書・入金・ステータス管理を中心に、業務アプリとして必要になりやすい 「一覧 / 検索 / 詳細 / 更新」 を一通り揃えたダッシュボードです。
Lite版として要件を絞りつつも、設計（データ構造・画面遷移・API）と実装の整合が伝わる構成を目指しています。

  • Backend：ASP.NET Core（Minimal API）
  • Frontend：Next.js / TypeScript / Tailwind
  • DB：MySQL（本番想定） / SQLite（ローカル開発用）
  • 対象：個人開発（業務想定）

※本ページは制作途中の内容を含みます。GitHub リポジトリは 2026/01/08 までに公開予定（限定公開想定）です。

実装状況

「どこまでできていて、何がこれからか」を明確にして、進捗と設計方針が伝わるように整理しています。

実装済み（現時点）

  • 認証/認可（ログイン・ロール制御）
  • 請求書：一覧/検索/詳細/作成/更新
  • 請求明細（InvoiceLines）：追加/更新/削除
  • 入金：一覧/検索/詳細/登録
  • ステータス（未払い/一部/完了 等）管理

※機能は「動作する状態」を基準に記載しています（UIのみ/未結線は実装済みに含めません）。

これから（実装予定）

  • 催促（リマインド）：送信履歴・テンプレ
  • 売上集計：月次/顧客別/ステータス別
  • 監査ログ：重要操作の追跡（管理者向け）
  • テスト拡充（API/サービス層）

「未実装」を明確にすることで、要件整理と優先順位付けの判断が伝わるようにしています。

設計方針

  • 関心の分離：API / Application / Domain / Infrastructure
  • DTO で入出力を固定し、内部実装の変更に強い構造へ
  • 一覧は標準で：検索条件・ページング・ソート

API一覧（設計・実装ベース）

カテゴリ Method Path 概要
Invoices GET /api/invoices 請求書一覧（検索/ページング）
Invoices GET /api/invoices/{id} 請求書詳細（明細含む）
Invoices POST /api/invoices 請求書作成
Invoices PUT /api/invoices/{id} 請求書更新
Payments GET /api/payments 入金一覧（検索/ページング/サマリー）
Payments GET /api/payments/{id} 入金詳細（割当込み）

※本表は「現時点の設計・実装に基づく一覧」です。公開までに命名・Path は調整する可能性があります。
